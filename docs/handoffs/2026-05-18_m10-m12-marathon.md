## 학습한 내용

### Multi-agent audit가 catch한 패턴

- **LVS_OWNERDATA selection-bit persistence**: virtual list-view가 `SetItemCountEx(0)` 후에도 LVIS_SELECTED 비트를 row index별로 내부 보관. 새 batch 도착 시 새 row가 옛 bit를 inherit. `ListView_SetItemState(-1, 0, LVIS_SELECTED | LVIS_FOCUSED)` broadcast가 root fix. Win32 docs는 "items beyond the new count are removed and their state is cleared"이라 명시하지만 새 count로 grow 시 새 row의 default state가 0인지 보장 없음 — 실측 Windows 10/11에서 persist 확인.
- **Subclass lifetime across InvokeCommand**: shell verb (Open With, Share)가 nested modal pump를 띄움. `TrackPopupMenuEx` 종료 후 `InvokeCommand` 전에 `RemoveWindowSubclass`하면 invoke 단계 menu pump가 forwarding을 잃음. SubclassGuard scope를 invoke 끝까지 확장 필수. L1/L2 모두 jointly 잡은 진짜 bug.
- **`lpVerb` ANSI-only / `lpVerbW = nullptr`**: 일부 shell extension이 `lpVerbW non-null + IS_INTRESOURCE(true)` 조합에서 string으로 오해해 dereference crash. `CMINVOKECOMMANDINFOEX`에서 `CMIC_MASK_UNICODE` 제거 + `lpVerb = MAKEINTRESOURCEA(idx)`만 set이 안전.

### Win32 multi-pane independence는 dispatch routing 단일 audit으로 풀린다

- 10+ 핸들러가 `pane_` / `listView_` hardcode였음. Explore agent의 dispatch map → L1+L2가 severity 분류 → 한 atom에서 일괄 fix. dispatch routing을 message source (`hdr->hwndFrom`)로 통일하는 invariant 한 줄이 dual-pane bug 4종 (LVN_ITEMCHANGED, GETDISPINFO, COLUMNCLICK, CUSTOMDRAW)을 동시에 해결.
- 헬퍼 `paneIndexFromListView(hwnd, &idx)` 추출 → 모든 핸들러 1-line 매핑. open-code loop 4 sites가 점진적으로 누적되었던 것을 batch review가 한 번에 잡아냄.

### Shell IDropTarget 위임 패턴은 외부 interop 공짜로 제공한다

- 자체 IDataObject 구현 안 함. `IShellFolder::GetUIObjectOf(IID_IDataObject)`로 shell이 만든 객체 그대로 사용. drop side도 `IShellFolder::CreateViewObject(IID_IDropTarget)` 위임. shell이 same-drive Move / cross-drive Copy / modifier (Ctrl/Shift/Alt) / conflict UI / progress dialog 자동 처리.
- 외부 Windows Explorer ↔ 우리 pane drop이 추가 코드 없이 동작. `RegisterDragDrop`만 set하면 됨.
- clipboard도 같은 패턴: `IDataObject` shell이 만들고 `CFSTR_PREFERREDDROPEFFECT`만 stamp. paste는 `IDropTarget`에 DragEnter → DragOver → Drop 시뮬레이션. paste(Move) 시 shell이 source 삭제까지 자동.

### `MainWindow` 1600줄 god-class signal

- M10 → M12 누적으로 dispatch hub MainWindow가 +900 LOC. Audit이 SOLID-SRP HIGH로 명시. `CutStateTracker`는 추출했지만 다음 후보:
  - `ClipboardCommandRouter` (`handleClipboardCopy/Paste` + `collectSelectedLeaves`).
  - `DragDropController` (`handleBeginDrag` + `dropTargets_[2]` lifecycle).
  - `AddressBarHost` (`addressBars_[2]` + subclass + popup wiring + dropdown CBN_DROPDOWN).
- 다음 milestone refactor에서 처리 권장. 동작 결함은 없으나 다음 기능 추가 비용을 낮춤.
